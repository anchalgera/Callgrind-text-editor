This example demonstrates a modern rich text editor for callgrind files.

What is a callgrind file?

A callgrind file is a profile data output from the Callgrind tool, which is part of the Valgrind instrumentation framework. It contains detailed information about a program's execution, including function calls, instruction counts, and cache behavior.

Types of information in callgrind files:

  • Function calls and their relationships (call graph)
  • Instruction counts
  • Cache misses (L1 and L2)
  • Branch prediction data
  • Source code line numbers
  • Object file information

How we are highlighting callgrind files:

Our text editor uses syntax highlighting to make callgrind files more readable. The highlighting includes:

  • Comments (lines starting with #) in gray
  • Header keywords (e.g., "version:", "events:") in purple
  • File specifications (fl=) in green
  • Function specifications (fn=) in blue
  • Numbers (including line numbers and event counts) in red
  • Cost lines (line number + event counts) with special formatting

Callgrind Example with Highlighting:

        # This is a comment line in callgrind file
        version: 1.0
        events: fl=main.cpp
        fn=foo
        100 300
        fn=bar
        150 400
